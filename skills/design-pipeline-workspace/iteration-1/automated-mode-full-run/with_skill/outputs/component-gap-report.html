Recommended Build Order

Based on reusability across the product, frequency of use, and dependency relationships.

  1. Status Tag / Badge -- Used on every screen, in every table row, and on dashboard cards. Foundation component.
  2. Button -- Used everywhere. Needs primary/secondary/ghost/success/small variants with icon support.
  3. Data Table -- Primary interface component. Depends on Status Tag and Button being available.
  4. Bulk Action Bar -- Companion to Data Table. Build together.
  5. Tabs -- Secondary navigation pattern needed across many pages.
  6. Toast Notification -- Global feedback mechanism.
  7. Error Banner -- Error state pattern.
  8. Search Input + Filter Chip -- Toolbar patterns, build together.
  9. Stat Card + Progress Bar -- Dashboard patterns, build together.
  10. Toggle Tabs, Confidence Indicator, Empty State -- Lower priority, build as needed.

Token Coverage

The prototype uses Software DS color tokens exclusively via CSS custom properties. All semantic tokens (--sds-bg-*, --sds-text-*, --sds-border-*, --sds-interactive-*, --sds-status-*, --sds-nav-*) are used correctly. No raw hex values bypass the token system. The color system is well-covered; the gap is in component patterns, not tokens.
