Recommendations for Next Iteration

  • Usability test the confidence threshold. Validate whether users prefer a fixed auto-approve threshold (e.g., 95%+) or prefer to review all suggestions. The "Approve all high confidence" button assumes users trust the threshold, which needs validation.
  • Add onboarding flow for first-time users. The empty state for data sources exists but the classification screen lacks a guided first-run experience. New users may not know where to start.
  • Add notification system for new data. The journey map identified that new unclassified data appearing without notification is a pain point. Consider email/in-app alerts when new fields are discovered during scheduled scans.
  • Test cross-source grouping with 100+ sources. The current design shows groups as cards, but users with hundreds of sources may need search/filter within the grouping view.
  • Consider keyboard shortcuts for power users. Maria Santos uses the tool 4-6 hours daily. Keyboard shortcuts for approve (A), reject (R), flag (F), and next field (down arrow) would significantly improve efficiency.
  • Add data owner notification in review workflow. The journey map showed data owners are pulled in without context. The review queue should allow tagging data owners with specific questions, not just flagging for security admin.
  • Build a classification progress tracking view. Maria's persona needs to track progress toward quarterly coverage goals. A personal progress tracker with targets and trends would be motivating.
