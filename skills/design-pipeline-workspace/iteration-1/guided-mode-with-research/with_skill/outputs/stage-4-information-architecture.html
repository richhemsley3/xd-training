User Flows

All proposed flows organized by type: happy path, edge case, error state, and fringe scenario.

Flow 1: Submit Access Request Happy Path

Data Engineer submits a new access request that is automatically routed to the correct approver.

Data Engineer Flow
Browse Catalog
→
Select Dataset
→
Click "Request Access"
→
Fill Justification
→
Auto-Route to Owner
→
Track Status
→
Access Granted

Flow 2: Approve Access Request Happy Path

Data Owner reviews and approves a request from their approval queue.

Data Owner Flow
Notification
→
Open Approval Queue
→
Review Request Context
→
Approve or Deny?
↓ Approve
Add Conditions (optional)
→
Auto-Provision
→
Done

Flow 3: Duplicate Request Detection Edge Case

System detects a matching pending request and surfaces it instead of creating a duplicate.

Data Engineer Flow
New Request Form
→
Select Dataset
→
Duplicate Check
→
Match Found?
↓ Yes
Show Existing Request
→
View Status

Flow 4: Batch Approval Edge Case

Data Owner with many pending low-risk requests can batch-approve them with guardrails.

Data Owner Flow
Approval Queue
→
Filter: Low Risk
→
Select All
→
Confirm Batch?
↓ Confirm
Validate Policies
→
Auto-Provision All
→
All Granted

Flow 5: Request Denied Error State

Data Owner denies a request with a reason. Requestor can amend and resubmit.

Cross-Persona Flow
Review Request
→
Approve or Deny?
↓ Deny
Provide Reason
→
Request Denied
→
Requestor Notified
→
Amend & Resubmit
→
Re-Route

Flow 6: No Approver Found Error State

Auto-routing cannot determine the correct approver. Falls back to admin escalation.

System Flow
Request Submitted
→
Auto-Route
→
Owner Found?
↓ No
No Owner Match
→
Escalate to Admin
→
Admin Assigns
→
Resume Flow

Flow 7: Policy-Based Auto-Approval Fringe

Low-sensitivity dataset with matching role policy triggers automatic approval with no human review.

System Flow
Request Submitted
→
Check Policies
→
Auto-Approve?
↓ Yes
Validate Role
→
Auto-Provision
→
Access Granted
→
Audit Log Entry

Flow 8: Periodic Access Review Fringe

Security Admin triggers a review cycle to certify or revoke existing access grants.

Security Admin Flow
Start Review
→
Select Scope
→
Generate Review List
→
Send to Owners
→
Owners Certify/Revoke
→
Execute Revocations
→
Review Complete

Flow Summary

All proposed flows with their type, primary persona, and research backing.

Flow Type Persona Research Backing
Submit Access Request Happy Path Data Engineer 40% time on workflows, no standard form
Approve Access Request Happy Path Data Owner 3-5 handoffs, requests lack context
Duplicate Detection Edge Case Data Engineer ~30% duplicate request rate
Batch Approval Edge Case Data Owner Batch approver pattern, overwhelmed owners
Request Denied Error State Cross-persona No feedback loop for denied requests
No Approver Found Error State System Approvals getting lost, unclear ownership
Auto-Approval Fringe System Inconsistent policies, over-permissioning
Periodic Access Review Fringe Security Admin Audit fire drill pattern, no audit trail
