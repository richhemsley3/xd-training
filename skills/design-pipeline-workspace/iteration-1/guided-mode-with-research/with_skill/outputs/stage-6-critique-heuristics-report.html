Pass 1 — Initial Evaluation

First pass identified 12 issues across critique and heuristic dimensions. All fixed before Pass 2.

Pass 1 · 12 findings
No confirmation after request submission H1: Visibility
Critical
After submitting a new request, the user is not shown a confirmation state. They have no immediate feedback that their request was received and routed.
Fix applied: Added a success confirmation banner at the top of the request detail screen after submission, showing the request ID and the approver it was routed to. Also added a toast notification pattern.
Screen: New RequestSeverity: CriticalFixed
No loading or processing states H1: Visibility
Major
Form submission, approval actions, and data loading have no intermediate states. Users cannot tell if their action is being processed.
Fix applied: Added loading spinner to submit and approve buttons during processing. Added skeleton loading states for table data and request detail content.
Screen: AllSeverity: MajorFixed
No undo/cancel for approval actions H3: User Control
Critical
Approve and Deny buttons execute immediately with no confirmation dialog. An accidental click on "Deny" cannot be reversed.
Fix applied: Added confirmation dialog for both Approve and Deny actions. Deny requires a reason before the dialog can be confirmed. Approve shows a summary of what will be granted.
Screen: Review RequestSeverity: CriticalFixed
Filter bar has no active state feedback H1: Visibility
Minor
When filter chips are clicked on the My Requests and Audit Log screens, there is no clear indication of which filter is active beyond the initial "All" state.
Fix applied: Active filter chip now uses blue-050 background with blue-750 text and a subtle border change. Added result count next to active filter.
Screen: My Requests, Audit LogSeverity: MinorFixed
Inconsistent back navigation pattern H4: Consistency
Major
Some detail screens use a text-based back link while others rely solely on the sidenav. The back link style varies between screens.
Fix applied: Standardized all detail screens to use a consistent breadcrumb pattern: "My Requests / REQ-2847" with the parent as a clickable link. Added aria-label for accessibility.
Screen: Request Detail, Review Request, DeniedSeverity: MajorFixed
Dataset selector lacks search in new request form H7: Flexibility
Major
The dataset dropdown is a simple select element. With hundreds of datasets, users cannot search or filter. This contradicts the research finding about needing a discoverable catalog.
Fix applied: Replaced the select with a searchable combobox that shows dataset name, schema, sensitivity level, and owner as the user types. Includes recent datasets shortcut.
Screen: New RequestSeverity: MajorFixed
No empty states for filtered views H9: Help Users
Minor
When filters return zero results on My Requests or Approval Queue, the table body is empty with no guidance.
Fix applied: Added empty state component with an icon, message, and suggested action for each filtered view (e.g., "No denied requests. All your requests have been approved or are pending.").
Screen: My Requests, Approval QueueSeverity: MinorFixed
Batch approval has no scope limit H5: Error Prevention
Major
The "Batch Approve (Low Risk)" button could approve dozens of requests without a clear count or confirmation. Research showed batch-approving under pressure is already a problem.
Fix applied: Button now shows count ("Batch Approve 4 Low Risk"). Clicking opens a confirmation dialog listing all requests to be approved, with a maximum batch size of 10. Requests above medium risk are excluded from batch selection.
Screen: Approval QueueSeverity: MajorFixed
No help text for risk meter H10: Help & Docs
Minor
The risk meter (colored bars) is not explained. New approvers may not understand what it represents or how risk is calculated.
Fix applied: Added tooltip on hover showing risk factors (sensitivity level, PII field count, requestor's existing access scope). Added a "How is risk calculated?" help link in the queue header.
Screen: Approval QueueSeverity: MinorFixed
Audit log pagination missing H7: Flexibility
Major
The audit log shows all events in a single scroll. For security admins reviewing months of activity, this becomes unmanageable.
Fix applied: Added date range picker filter and pagination (25/50/100 per page) with total count. Also added a "Jump to date" shortcut.
Screen: Audit LogSeverity: MajorFixed
No notification preferences visible H3: User Control
Minor
Users receive notifications but cannot control which notifications they get (email vs. in-app, frequency). Settings link exists but preferences are not surfaced.
Fix applied: Added notification dropdown from the bell icon showing recent notifications with a "Manage preferences" link. Settings page includes granular notification controls.
Screen: All (header)Severity: MinorFixed
Denied state lacks clear next steps H9: Help Users
Minor
While the denied screen shows a reason, the "Amend & Resubmit" button does not pre-populate the original form data. Users would need to re-enter everything.
Fix applied: "Amend & Resubmit" now opens the new request form pre-filled with the original request data, with the justification field highlighted for editing. The original denial reason is shown as context.
Screen: Denied StateSeverity: MinorFixed

Pass 2 — Re-evaluation

Second pass on updated designs found 4 additional issues introduced or revealed by the fixes.

Pass 2 · 4 findings
Confirmation dialog blocks keyboard navigation H7: Flexibility
Major
The new confirmation dialogs for approve/deny do not trap focus properly and lack keyboard shortcuts for confirm/cancel actions.
Fix applied: Added focus trap to modal dialogs. Enter confirms, Escape cancels. Tab cycles through focusable elements within the dialog.
Screen: Review RequestSeverity: MajorFixed
Searchable combobox lacks keyboard support H4: Consistency
Major
The new dataset search combobox does not support arrow key navigation through results or Enter to select.
Fix applied: Added full ARIA combobox pattern with arrow key navigation, Enter to select, and Escape to close. Results are announced to screen readers.
Screen: New RequestSeverity: MajorFixed
Breadcrumb truncation on narrow viewports H8: Aesthetic Design
Minor
The new breadcrumb pattern overflows on narrow viewports, pushing content off-screen.
Fix applied: Added text-overflow: ellipsis to breadcrumb items with a max-width constraint. On mobile, breadcrumb collapses to show only the parent link.
Screen: All detail screensSeverity: MinorFixed
Toast notification overlaps with batch confirmation H8: Aesthetic Design
Minor
If a batch approval triggers a toast while the confirmation dialog is still closing, the two overlap visually.
Fix applied: Toast notification now queues behind any open dialog and displays after dialog closes. Added z-index management for layered UI elements.
Screen: Approval QueueSeverity: MinorFixed
